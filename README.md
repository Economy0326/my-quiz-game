# 나만의 퀴즈 게임

## 프로젝트 개요
Python 콘솔 기반으로 동작하는 퀴즈 게임입니다.

사용자는 메뉴를 통해 퀴즈를 풀고, 새로운 퀴즈를 추가하고, 등록된 퀴즈 목록과 최고 점수를 확인할 수 있습니다.
또한 프로그램 종료 후 다시 실행해도 `state.json` 파일을 통해 퀴즈 데이터와 최고 점수가 유지되도록 구현했습니다.

## 퀴즈 주제 선정 이유
이번 프로젝트에서는 Python 기초 문법과 프로그래밍 상식을 주제로 퀴즈를 구성했습니다.

직접 퀴즈를 만들고 풀어보는 과정에서 Python의 기본 문법, 자료형, 조건문, 반복문, 클래스 개념을 자연스럽게 복습할 수 있다고 생각해서 이 주제를 선택했습니다.

## 실행 방법

### 1. 프로젝트 폴더로 이동
    cd my-quiz-game

### 2. 프로그램 실행
    python main.py

또는 환경에 따라 아래 명령어로 실행할 수 있습니다.

    py main.py

## 기능 목록
- 퀴즈 풀기
- 퀴즈 추가
- 퀴즈 목록 보기
- 최고 점수 확인
- `state.json` 파일 저장 및 불러오기
- 잘못된 입력 처리
- `KeyboardInterrupt`, `EOFError` 예외 처리

## 클래스 구조

### Quiz 클래스
개별 퀴즈 1개를 표현하는 클래스입니다.

포함 정보:
- 문제
- 선택지 4개
- 정답 번호

주요 역할:
- 퀴즈 내용 출력
- 사용자 정답 확인
- JSON 저장용 딕셔너리 변환
- JSON 데이터에서 Quiz 객체 복원

### QuizGame 클래스
게임 전체 흐름을 관리하는 클래스입니다.

포함 정보:
- 전체 퀴즈 목록
- 최고 점수

주요 역할:
- 메뉴 출력
- 퀴즈 풀기
- 퀴즈 추가
- 퀴즈 목록 출력
- 점수 확인
- 파일 저장 / 불러오기
- 프로그램 실행 흐름 제어

## 데이터 파일 설명

### state.json 경로
프로젝트 루트 경로에 생성됩니다.

    my-quiz-game/state.json

### 역할
- 등록된 퀴즈 목록 저장
- 최고 점수 저장

### 예시 스키마
    {
      "quizzes": [
        {
          "question": "Python의 창시자는?",
          "choices": [
            "Guido van Rossum",
            "Linus Torvalds",
            "James Gosling",
            "Bjarne Stroustrup"
          ],
          "answer": 1
        }
      ],
      "best_score": 3
    }

## 입력 예외 처리
프로그램에서는 아래와 같은 잘못된 입력을 처리하도록 구현했습니다.

- 빈 입력
- 숫자가 아닌 입력
- 허용 범위를 벗어난 숫자 입력
- `Ctrl + C` 입력
- 입력 스트림 종료(`EOFError`)
- `state.json` 파일이 없거나 손상된 경우

파일이 없으면 기본 퀴즈 데이터를 사용하고,
파일이 손상되었으면 안내 메시지를 출력한 뒤 기본 데이터로 복구하도록 구현했습니다.

## Git 작업 내역 요약
이번 프로젝트에서는 Git을 사용해 기능 단위로 커밋을 나누어 작업했습니다.

예시:
- `Init: 프로젝트 초기 파일과 메뉴 구조 생성`
- `Feat: Quiz 클래스 출력과 정답 확인 메서드 추가`
- `Feat: 기본 퀴즈 데이터 5개 추가`
- `Feat: 퀴즈 출제와 점수 계산 기능 구현`
- `Feat: 퀴즈 추가 기능 구현`
- `Feat: 등록된 퀴즈 목록 조회 기능 추가`
- `Feat: 최고 점수 확인 기능 추가`
- `Feat: state.json 저장 및 불러오기 기능 구현`
- `Fix: 비정상 종료와 입력 예외 처리 추가`
- `Docs: README 작성`

또한 브랜치를 생성하고 병합했으며, `clone`과 `pull`도 실습했습니다.

## 느낀 점
이번 프로젝트를 통해 Python 기본 문법만 아는 것과, 직접 동작하는 프로그램을 끝까지 만드는 것은 다르다는 점을 느꼈습니다.

특히 클래스 분리, 파일 저장, 예외 처리, Git 브랜치 관리까지 경험하면서 단순 문법 연습보다 훨씬 실제적인 개발 흐름을 익힐 수 있었습니다.
